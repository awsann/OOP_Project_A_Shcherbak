# Project A: ВІДНОШЕННЯ МІЖ КЛАСАМИ:АСОЦІАЦІЯ, АГРЕГАЦІЯ, КОМПОЗИЦІЯ, РЕАЛІЗАЦІЯ

## Мета роботи
Мета практичної роботи – спроектувати систему управління бібліотекою відеоігор з використанням принципів об’єктно-орієнтованого програмування та різних типів зв’язків між класами:
- Асоціація  
- Агрегація  
- Композиція  
- Реалізація інтерфейсів  

Також робота включає створення каркасу проекту, розробку unit-тестів та використання GitHub-репозиторію для контролю версій.
## Опис предметної області
Система призначена для:
- Організації та обліку колекції відеоігор;  
- Відстеження ігрових сесій користувачів;  
- Управління досягненнями в іграх.
## Unit-тести
Реалізовано 15 unit-тестів, які перевіряють:
- Конструктори класів
- Реалізацію інтерфейсу `IPlayable`
- Додавання/видалення ігор
- Створення ігрових сесій
- Управління досягненнями
- Отримання інформації про колекцію ігор

Всі тести очікувано завершуються **Failed**, оскільки методи класів ще не реалізовані.  
Підхід відповідає принципам **TDD**.
# Висновки
Спроектовано повноцінну модель предметної області для системи управління бібліотекою відеоігор.
Створено класи з усіма необхідними зв’язками (асоціація, агрегація, композиція, реалізація інтерфейсів).
Unit-тести підтверджують правильність вибору архітектури та готовність до подальшої реалізації функціоналу.
